# Prompt Template Structure

This document describes the structure of crash-fix prompts generated by action/build-prompt.sh.

## Format

Prompts are constructed in Markdown format with the following structure:

### Required Sections
- **Crash Information**: Signature, app version (always present)
- **Stack Trace**: Full stack trace text
- **Task**: Instructions for the agent to investigate and fix

### Optional Sections
- **Device Info**: Device make/model and Android version (if provided)
- **Occurrences**: Number of users affected (if provided)
- **Create Time**: ISO 8601 timestamp (if provided)

## Example

```markdown
# Crash Fix Task

## Crash Information
**Signature:** NullPointerException: Attempt to invoke virtual method 'int java.lang.String.length()' on a null object reference
**App Version:** 1.2.3
**Device:** Pixel 4 Android 12
**Occurrences:** 142
**Time:** 2026-05-19T10:30:00Z

## Stack Trace
java.lang.NullPointerException: Attempt to invoke virtual method 'int java.lang.String.length()' on a null object reference
  at com.example.app.MainActivity.onCreate(MainActivity.java:45)
  at android.app.Activity.performCreate(Activity.java:8080)
  ...

## Task

Investigate this crash and propose a minimal fix. Your response should:
1. Explain the root cause
2. Identify the file(s) that need changes
3. Provide a patch or code changes
4. Explain why this fix resolves the crash

Scope changes **only** to files implicated by the stack trace. Do not refactor or improve unrelated code.
```

## Prompt Building Rules

1. **Markdown format**: All prompts are Markdown to ensure readability and structure
2. **Optional field omission**: If an optional field (device-info, occurrence-count, create-time, subtitle) is not provided, it is omitted from the prompt
3. **Stack trace**: Always present; if not provided by the caller, a placeholder message indicates missing trace
4. **Scoping instruction**: Always included to ensure the agent limits changes to crash-related files only

## Variables Populated by action/build-prompt.sh

- `SIGNATURE`: Exception class and brief description
- `APP_VERSION`: Version of the app at crash time
- `DEVICE_INFO`: Optional device specification
- `OCCURRENCE_COUNT`: Optional count of affected users
- `CREATE_TIME`: Optional ISO 8601 timestamp
- `STACK_TRACE`: Full stack trace (required)

All variables are read from environment variables set by the GitHub Action and inserted into the prompt template.
